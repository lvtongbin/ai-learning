## 目标

为 Module 03 中 `### Skill 怎么写` 补充内容。目标读者是有经验的开发者，已经掌握 Skill 语法，需要提升判断力和协作效率。

## 核心思想

写什么 > 怎么写。格式让 AI 生成，判断力来自人。

## 内容结构

### 1. 核心观点（1段）

点题：Skill 的质量瓶颈不在格式，在判断力。格式可以让 AI 写，但"这个场景值不值得固化成 Skill"只有人能判断。

### 2. 判断标准（3个维度）

每个维度一句话，三个都是 yes 才值得写：

- **重复性**：这个流程你是否已经在脑子里执行过 3 次以上？
- **专业性**：这里有你的团队/项目特有的知识，AI 默认不知道？
- **可描述性**：你能用一段话说清楚"什么时候触发、做什么"？

### 3. 协作流程（3步）

1. 人定方向：描述场景、触发条件、期望输出
2. AI 生成草稿：根据描述生成完整 Skill 文件
3. 人审核两处：`description`（触发条件准不准）+ 正文流程（关键步骤有没有遗漏）

### 4. 收尾（1句）

审核 `description` 比审核正文更重要——触发条件错了，再好的正文也没用。

## 写作风格

与 Module 03 其他章节保持一致：简洁、有观点、不废话。
